## What this demonstrates

This project analyses public Spotify user feedback to understand why meaningful music discovery breaks down and why listeners fall back into familiar/repetitive listening.

The live workflow demonstrates:

1. Fresh public data collection from Google Play and Reddit
2. Normalisation of multi-source review data
3. AI analysis of discovery-related complaints and themes
4. Evidence-backed reporting with source counts and sample quotes
5. A chatbot interface for querying the frozen research dataset

## Architecture

```text
n8n Form Trigger
  -> Apify Google Play scrape
  -> Apify Reddit scrape
  -> Normalise Google Play reviews
  -> Filter and normalise Reddit posts
  -> Merge records
  -> Prepare records for AI
  -> Groq LLM analysis
  -> Form Ending report
```

The chatbot workflow uses:

```text
n8n Chat Trigger
  -> AI Agent
  -> Spotify Review Analytics Tool
  -> Supabase Vector Store
  -> Google Gemini Embeddings
  -> Groq Chat Model
```

## Final research dataset

The main research dataset is frozen separately from the public demo workflow.

* Raw records scraped: 350
* Stored and classified records: 312
* Google Play records: 300
* Reddit records: 12
* Discovery-related records: 37
* Discovery-related share: 11.9%

The public live demo fetches a fresh sample on each run and should not be treated as the complete research dataset.

## Demo modes

The public form supports controlled run sizes to protect API quota:

* Quick test: 1 Google Play review + 1 Reddit post
* Demo: 15 Google Play reviews + 2 Reddit posts
* Full analysis: 300 Google Play reviews + 50 Reddit posts

Demo mode is recommended for evaluators.

## Limitations

* Public demo results vary because they fetch live data.
* Demo mode uses a small sample to protect free-tier API quotas.
* The main conclusions in the deck are based on the frozen 312-record classified dataset, not on a single public demo run.
* Reddit data after filtering can be smaller than the requested scrape size because irrelevant posts are removed.
